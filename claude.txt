# 多平台产品推广文章生成器

## 项目目标
根据不同平台，生成多样性的产品推广文章。产品为 bewild.ai（AI 代订阅平台）。

## 技术栈
- 后端：FastAPI + Python（asyncio）
- 前端：Jinja2 模板 + 原生 JS + 手写 CSS
- API：DeepSeek（通过 OpenAI SDK 兼容接口）
- 配置：YAML（block scalar `|` 保留格式）
- 无数据库，无状态管理，文件系统持久化

## 项目结构（实现状态）

```

│       ├── zhihu.yaml          # 知乎风格 + emphasis（可为空）
│       ├── xiaohongshu.yaml    # 小红书风格 + 违禁词强调 emphasis
```

## Prompt 组装方式

直接拼接，不使用占位符。顺序：

```
diversity_prompt + "---" + platform_style + "---" + template(可选) + "---" + emphasis(可选)
```

### diversity_prompt 的构建（config_loader.build_diversity_prompt）

从 diversity.yaml 中每类随机抽取一项拼成：
```
{persona}

{ai_level}

{scene}

{angle}

主要围绕关键词：{main_keyword}
辅助关键词：{auxiliary_keyword}

【产品信息】
{product_info}
```

- 前端通过勾选控制每类的可选范围（selection），传 {"persona": [...], "scene": [...], ...} 给后端
- 后端 `_pick(items, selection)` 只在勾选范围内随机抽取
- 若某类前端全不选或未传，则从全部候选中抽取
- 每次生成独立抽取，各文章随机组合不同

### platform_style（config/platforms/*.yaml）
- `style_prompt`：平台风格要求，使用 YAML block scalar `|`
- `emphasis`：末尾追加的强调内容。小红书用此字段重复违禁词提醒，其他平台可为空

### template（templates/*.md）
- 可选，前端下拉选择，内容插入到 platform_style 之后

## 多样性配置（config/diversity.yaml）

6 个可随机抽取的列表类字段：
- `persona`（6项）：人物定位（产品经理/大学生/自由职业者/职场妈妈/创业者/考研党）
- `scene`（6项）：写作场景（通勤/深夜/咖啡馆/会议后/假期前/帮朋友后）
- `ai_level`（3项）：AI熟悉度（老手/新手/门外汉）—— 影响用词专业度和产品介绍深度
- `angle`（6项）：论点切入角度（省时间/对比/场景/改观/技巧/设计思路）
- `main_keywords`（6项）：主关键词
- `auxiliary_keywords`（6项）：辅助关键词

2 个固定字段（不随机抽取，全文写入 prompt）：
- `product_info`：产品背景（官方订阅 + bewild 代订阅 + 价格约定 + 禁止行为）
- `forbidden_words`（41项）：违禁词列表，用于生成后检测

## 平台配置（config/platforms/*.yaml）

每个平台：
- `name`：平台中文名
- `style_prompt`：平台风格要求（block scalar `|`）
- `emphasis`：末尾强调内容。小红书有违禁词强调；知乎/X/Bilibili 为空字符串

## API 调用（api.py）

- 模型：通过 `DEEPSEEK_MODEL` 环境变量配置（默认 `deepseek-chat`）
- 超时：单次请求 120 秒
- 重试：最多 3 次，指数退避（2s / 4s / 8s）
- 并发：asyncio.Semaphore(5)，全局单例
- 客户端：AsyncOpenAI，全局单例
- temperature=0.8

## 生成流程（generate.py）

1. 加载 diversity 配置 + 平台配置 + 模板
2. 创建输出文件夹 `output/{platform}_{timestamp}/`
3. 并发执行 N 篇（Semaphore(5)）：
   a. build_diversity_prompt(diversity, selection) → 随机抽取拼 prompt
   b. _assemble_prompt() → 合并 diversity + 平台风格 + 模板 + emphasis
   c. generate_text() → 调 API
   d. check_banned_words() → 扫描违禁词
   e. 保存 `{index:03d}.md`（文章 + 元信息头部）+ `{index:03d}_prompt.md`（完整 prompt）
   f. 写入日志
4. 通过 SSE 流式推送进度事件给前端

生成状态有三种：ok（完成）、banned（命中违禁词）、error（API 异常）。

## 违禁词检测（generate.py check_banned_words）

- 生成后扫描正文，子串匹配 forbidden_words 列表
- 命中则状态标记为 `banned`，文章仍保存，头部注明命中词
- 前端日志紫色显示

## 配置保存（main.py _save_yaml_text）

- 先 `yaml.safe_load()` 验证 YAML 合法性
- 验证通过后直接 `write_text()` 写入原始文本
- 不经过 `yaml.dump()` 往返，保留用户格式和注释（包括 block scalar `|`）

## 本地前端

功能：
- Tab 切换编辑各配置文件（多样性/知乎/小红书/X/Bilibili），直接编辑 YAML 原文
- 关键词选择区：每类以 chips 渲染，勾选/取消控制随机抽取范围，全选/全不选按钮
- 一键生成：选平台 + 篇数 + 模板，POST 请求 SSE 流式接收进度
- 生成日志：页面加载历史日志，生成时实时更新

## 不做的事项
- 不做文章去重 / 相似度检测
- 暂不编写测试
- 不做模型 fallback 链
- 不做按 angle+audience 的文件夹分层
- 不做标题关键词 / 正文关键词分离
- 不做各类别均匀分配的 rebalancing
